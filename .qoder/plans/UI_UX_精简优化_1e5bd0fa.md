# colM1_Chn.sh UI/UX 精简优化

## 问题清单与对应改法

### 1. 启动噪音过多（最影响体感）

**现状：** 启动时连续输出 4 段信息
```
主函数横幅
↓ load_seed_from_args: 路径/编译器映射回显
↓ initialize: "应用编译器映射配置..." + 每条 op 的设置回显
↓ "✅ 初始化完成！" + 脚本位置 + 源文件目录
↓ main_interface 再打一遍横幅
```

**改法：**
- `load_seed_from_args()` 的参数回显：用种子启动时只在最后 `initialize()` 汇总显示一次，取消每个参数单独 echo
- `initialize()` 删除 "✅ 初始化完成！" 和 "📄 脚本位置" 两行（用户不关心）；源文件目录保留，但挪到主横幅下方合并显示
- `main_interface()` 的 "主界面" 横幅 + 使用说明文字块 整体删除，内容已在 main() 横幅里涵盖
- **效果：** 启动只显示 1 个横幅 + 1 行源目录路径 + 提示符

---

### 2. 执行时前后两个框（最冗余）

**现状：**
```
┌─────────────────────────────┐
│        执行 test.c          │   ← 框1
│ 语言: c | 编译器: clang     │
└─────────────────────────────┘
🔨 编译 C 文件...
✅ 编译成功: ./test
🏃 运行程序...
--- 程序输出 ---
🗑️ 已删除编译产物
┌─────────────────────────────┐
│        执行结束             │   ← 框2（完全多余）
└─────────────────────────────┘
```

**改法：**
- 删除执行前的整个 3 行框，替换为 1 行简洁信息：`🔨 [c/clang] test.c`
- 删除执行后的 "执行结束" 框，无替代（程序输出结束即为结束）
- 删除完产物的 `🗑️ 已删除编译产物` 提示（静默删除即可，用户不需要知道）
- **效果：** 执行时只多 1 行前置信息，其余全是程序自己的输出

---

### 3. 框宽不一致

**现状：** `show_help()` 用 42 宽框，其余用 31 宽框，视觉断层

**改法：** 统一所有框为 **36 宽**（兼顾中文字符，不会太宽也不裁内容）

---

### 4. 安装提示冗余代码

**现状：** `execute_file()`、`apply_compiler_language_pairs()` 各有一套完整 case 块，共约 60 行，与 `get_install_hint()` 三份内容完全相同

**改法：** 两处均改为调用 `get_install_hint()`：
```bash
# execute_file() 中
echo "   💡 建议安装: $(get_install_hint "$compiler")"

# apply_compiler_language_pairs() 中
echo "   💡 建议安装: $(get_install_hint "$compiler")"
```
删除两个独立 case 块，约减少 40 行冗余代码

---

### 5. vls 双重收集文件（逻辑bug）

**现状：** `vls()` 函数末尾已写入全局 `vls_files`，但 `main_interface()` 调用 vls 后又重新 find 一遍

**改法：** 删除 `main_interface()` 里 `vls` 分支中的重复 find 块（约 8 行），直接使用 `vls()` 已设置的全局 `vls_files`。注意：`main_interface()` 中的 `local vls_files=()` 声明需改为不带 `local`（否则作用域隔离导致 vls() 写的值不可见）

---

### 6. 死代码清理

删除两行无用声明：
```bash
declare -A extension_commands     # 删除
declare -A extension_files        # 删除
```
